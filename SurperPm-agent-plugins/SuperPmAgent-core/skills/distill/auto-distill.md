# Auto-Distill Steps

Triggered by stop.py hook after /goal completes. Extracts reusable skills and knowledge from the current conversation.

## Steps

1. **Analyze the completed conversation**
   - Review the full conversation transcript (available in current context)
   - Identify what the goal was and what was accomplished
   
2. **Get the git diff for this goal**
   
   **Use script**: `scripts/get-goal-diff.sh`
   
   ```bash
   ./SuperPmAgent-core/skills/distill/scripts/get-goal-diff.sh /tmp/goal-diff.txt
   ```
   
   **Manual steps** (if script not available):
   
   **CRITICAL**: Use the commit recorded at goal start, NOT `git diff HEAD`.
   
   - Read start commit: `cat /tmp/goal-start-commit.txt`
   - Get diff: `git diff ${START_COMMIT} HEAD > /tmp/goal-diff.txt`
   - Fallback (if file missing): `git diff HEAD~1 HEAD`
   
   **Why this matters**:
   - `git diff HEAD` → compares working directory to HEAD (may be empty if committed)
   - `git diff HEAD~1` → assumes only 1 commit (may be wrong if multiple commits)
   - `git diff <start-commit> HEAD` → **correct**, shows ALL changes during this goal

3. **Identify candidate artifacts**
   - A reusable procedure or pattern → `kind=skill`
   - An architecture decision / code fact → `kind=knowledge`
   - A CI failure recovery pattern → `kind=skill, subkind=self-heal`

4. **Generate notes.md from conversation** (NEW - CRITICAL)

   **IMPORTANT**: notes.md MUST be dynamically generated from conversation.md, NOT manually written.

   ```markdown
   # Step 4a: Analyze conversation.md

   Read `knowledge/sessions/<session-name>/conversation.md` and extract:

   1. **Goal Summary**: What was the user trying to achieve?
   2. **Key Decisions**: What architectural/technical decisions were made?
   3. **Knowledge Candidates**: Patterns, conventions, or facts that should be distilled
   4. **Potential Skills**: Reusable procedures or workflows

   # Step 4b: Generate notes.md

   Create `knowledge/sessions/<session-name>/notes.md` with this structure:

   ```yaml
   ---
   session: <session-name>
   confidence: 0.8
   confidence_reason: "AI extracted from conversation"
   created: YYYY-MM-DD
   last_accessed: YYYY-MM-DD
   access_count: 0
   ttl_days: 90
   status: active
   source: session/<session-name>
   ---

   # Session Notes

   **Auto-generated by AI from conversation.md**  
   **Generated**: YYYY-MM-DD HH:MM

   ## Goal Summary

   <1-2 sentence summary of what the user wanted>

   ## Key Decisions

   1. **Decision 1**: Description
   2. **Decision 2**: Description

   ## Knowledge Candidates

   ### Candidate 1: <Title>

   **Type**: foundation | convention | context  
   **Confidence**: 0.X (reason)  
   **Keywords**: keyword1, keyword2, keyword3  
   **Business Area**: <area> (keywords match "<keywords>")  
   **Target**: `knowledge/domain/<area>/<type>/<slug>.md`  
   **Content**: <1-sentence description>  
   **Rationale**: <why this is valuable>

   ## Potential Skills

   ### Skill 1: <Title>

   **Type**: skill (coding | business)  
   **Confidence**: 0.X (reason)  
   **Keywords**: keyword1, keyword2  
   **Target**: `SuperPmAgent-<type>/skills/<slug>/SKILL.md`  
   **Content**: <1-sentence description>  
   **Rationale**: <why this is reusable>
   ```

   **CRITICAL Rules**:
   - The `Target` paths in Knowledge Candidates are **suggestions**, not final
   - AI MUST validate each target path using the Business Area Classification decision tree (SKILL.md Section 4)
   - If the suggested path is wrong, AI MUST correct it before generating files

5. **Decide target path**

   | Kind | Flavor | Target |
   |------|--------|--------|
   | skill | coding | `SuperPmAgent-coding/skills/<slug>/SKILL.md` |
   | skill | business | `SuperPmAgent-business/skills/<slug>/SKILL.md` |
   | skill | self-heal | `SuperPmAgent-coding/skills/fixes/<slug>/SKILL.md` |
   | knowledge | — | `knowledge/domain/<area>/<slug>.md` |

6. **Check for duplicates**
   
   **IMPORTANT**: Follow the dedup algorithm in SKILL.md (Section 3).
   
   For EACH artifact to be created:
   1. Extract keywords (title + first 3 headings + key terms)
   2. Grep target directory
   3. Calculate similarity (headings 40% + code 40% + keywords 20%)
   4. Apply decision matrix from SKILL.md
   
   Log results:
   - "Checked for duplicates: found N similar files, highest similarity X% → action"
   - If similarity > 70%, skip this artifact
   
   If updating existing file:
   - Preserve original frontmatter (name, created date)
   - Update description if significantly improved
   - Increment access_count by 1

7. **Generate the artifact**
   - Skills: create SKILL.md with proper frontmatter (`name`, `description` ≤ 1024 chars)
   - Knowledge: create .md file using the unified frontmatter template (see `knowledge/domain/INDEX.md`)
   - Set initial `confidence` based on source:
     - **0.8** for auto-distill (has git diff, saw actual changes)
     - **0.9** if user explicitly confirmed during conversation
     - **0.6** if inferred only from conversation (no code changes)

8. **Update ALL affected INDEX.md files**
   
   **IMPORTANT**: Follow the INDEX.md checklist in SKILL.md (Section 4).
   
   For each file created/modified/deleted, update the relevant INDEX.md.

9. **Update notes.md with Distillation Status**

   **CRITICAL**: After generating all artifacts, update notes.md to mark distillation status.

   Read the existing notes.md and insert a "Distillation Status" section **after the frontmatter**:

   ```markdown
   ---
   (existing frontmatter)
   ---

   # Distillation Status

   **Distilled**: YYYY-MM-DD  
   **Mode**: auto-distill  
   **Artifacts Created**: N

   | File | Type | Confidence |
   |------|------|-----------|
   | `knowledge/domain/...` | foundation | 0.9 |
   | `SuperPmAgent-coding/skills/.../SKILL.md` | skill | 0.8 |

   **INDEX.md Updated**:
   - [x] `knowledge/domain/INDEX.md`
   - [x] `knowledge/domain/<area>/INDEX.md`

   ---

   # Session Notes

   (existing content preserved)
   ```

   **IMPORTANT**: 
   - DO NOT delete or modify the existing content
   - ONLY insert the Distillation Status section after frontmatter
   - Preserve all Knowledge Candidates and other sections

10. **Open PR**
   
   **Follow the PR creation workflow from SKILL.md (Section 6).**
   
   **Mode-specific config**:
   - Branch: `distill/auto-<sanitized-session>` (use sanitized session name)
   - Commit: `distill(auto): extract <kind> from goal <goal-summary>`
   - PR title: `distill/auto: <slug>`
   
   **Execute the shared workflow** (defined in SKILL.md):
   1. Pre-flight Checks (check remote, gh CLI, auth, changes)
   2. Branch Creation (handle conflicts - check if branch exists)
   3. Commit and Push (with error handling - catch push failures)
   4. Create PR (with manual fallback - if gh fails, provide manual steps)
   5. Completion Summary (output PR URL and branch name)
   
   **IMPORTANT**: 
   - Create ONE PR for ALL artifacts (skills + knowledge + INDEX.md updates)
   - Do NOT create separate PRs
   
   ### Pre-flight Checks
   
   **Use script**: `scripts/create-pr.sh` (handles all pre-flight checks automatically)
   
   The script checks:
   - Remote 'origin' configured
   - gh CLI installed
   - GitHub authenticated
   - Changes exist to commit
   
   If checks fail, script will exit with guidance.
   
   ### PR Body Template
   
   ```markdown
   # Auto-Distill: <goal-summary>
   
   ## Source
   - Session: `<session-name>`
   - Goal: `<goal-text>`
   - Completed: `<YYYY-MM-DD HH:MM>`
   
   ## Extracted Artifacts
   
   ### Skills (N)
   | File | Type | Rationale |
   |------|------|----------|
   | `SuperPmAgent-*/skills/<slug>/SKILL.md` | coding/business | Why this is reusable |
   
   ### Knowledge (N)
   | File | Type | Confidence |
   |------|------|-----------|
   | `knowledge/domain/<area>/<slug>.md` | foundation/convention | 0.6-1.0 |
   
   ## Quality Gates
   - [ ] Deduplication checked (followed SKILL.md algorithm)
   - [ ] All SKILL.md descriptions ≤ 1024 chars
   - [ ] All INDEX.md files updated (checklist in SKILL.md)
   - [ ] No trivial content extracted
   
   ## Review Notes
   <Any caveats or suggestions for reviewer>
   ```
   
   ### Error Handling
   
   **Script handles errors automatically**:
   - Push failures: Script catches and reports
   - PR creation failures: Manual fallback provided
   - Pre-flight failures: Clear error messages
   
   If script fails, it will output:
   - What failed
   - Why it failed
   - How to fix it
